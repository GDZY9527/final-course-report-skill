# 前端质量

[English](frontend_quality.md) | [中文](frontend_quality.zh-CN.md)

## 强制标准

除纯硬件课程外，每个项目的软件部分必须包含完整、真实、可运行的前端。单纯截图、空白 HTML 或简陋表单都不合格。

## 前端必备能力

必须包含：

- 主工作台或仪表盘
- 核心功能操作区
- 真实项目数据或结果展示
- 加载、成功、空状态和错误反馈
- 至少一个有意义的交互闭环
- 与课程项目相关的指标、日志、来源、预测、记录或测试结果
- 清晰的启动说明
- 适合插入报告的截图

## UI 质量

前端应达到答辩演示和报告截图所需的专业程度：

- 信息层级清晰
- 桌面端和常见移动宽度下布局合理
- 文本不溢出
- 元素不混乱重叠
- 有可见交互状态
- 中文文本清晰可读
- 视觉风格克制、专业

避免只做装饰性页面。首屏应展示真实系统，而不是营销式落地页。

## 集成方式

前端必须通过以下方式之一反映真实项目行为：

- 后端 API 调用
- 本地服务路由
- 导出的 JSON/CSV 结果文件
- 基于真实结果数据生成的确定性静态预览

在 README 和 `results/report_notes.md` 中说明集成方式。
