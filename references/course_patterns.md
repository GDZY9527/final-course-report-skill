[English](course_patterns.md) | [中文](course_patterns.zh-CN.md)

## C++ Object-Oriented Programming

Use as a later optimized example:

- C++17 core project
- class-based design
- file persistence
- tests and run logs
- class diagram and system flow
- complete frontend management page

Pattern: course theory plus an object-oriented system with real test results and frontend visualization.

## LLM Application Development

Use as a later optimized example:

- RAG or knowledge-base QA project
- API path plus offline fallback
- retrieval metrics
- frontend QA page
- source snippets and report notes

Pattern: intended model/API route plus reproducible local fallback when environment is limited.

## Natural Language Processing

Use only as an early historical example for:

- theory chapter structure
- model evaluation metrics
- fallback explanation
- confusion matrix and prediction examples

Do not copy its missing-frontend project shape. New NLP projects must include full frontend/backend, such as a model service API and sentiment-analysis dashboard.

## General Adaptation

For a new course:

1. Identify core course concepts.
2. Choose a small but complete project that exercises those concepts.
3. Include frontend/backend unless purely hardware-only.
4. Produce real metrics and screenshots.
5. Write Chapter 3 from generated project evidence.
