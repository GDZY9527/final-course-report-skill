# 正文表达质量与 AI 痕迹控制

[English](prose_quality.md) | [中文](prose_quality.zh-CN.md)

## 目标

报告正文应像学生完成的课程报告，而不是带有提示词痕迹、术语堆砌或自动生成说明的草稿。技术术语可以严谨，但不能到处出现提示性括号、解释性括号和“此处/例如/说明/可替换”等痕迹。

## 括号使用规则

允许：

- 少量关键术语首次出现时给出英文全称或缩写，例如“潜在语义分析（Latent Semantic Analysis, LSA）”。
- 必要的公式编号、图表编号、文献引用编号。

限制：

- 同一术语只在首次出现时给英文解释，后文直接使用中文术语或缩写。
- 不要连续堆放多个英文括号解释。
- 不要用括号写写作提示、生成提示或自我说明。
- 一篇完整报告中的英文术语括号通常不超过 10-15 个，除非课程明确要求双语术语。

禁止：

- “（此处补充……）”
- “（可根据需要……）”
- “（例如……）”反复出现
- “（说明：……）”
- “（待完善/待补充）”
- “（用于展示/用于说明）”
- 明显像提示词残留的括号内容
- 大量百科式术语堆砌，例如一节中连续出现 `Hidden Markov Model, HMM`、`Conditional Random Fields, CRF`、`Large Language Model, LLM`，但没有和项目实现或课程分析直接关联

改写示例：

- 不推荐：`隐马尔可夫模型（Hidden Markov Model, HMM）、最大熵模型和条件随机场（Conditional Random Fields, CRF）在序列标注中被广泛使用。`
- 推荐：`早期序列标注通常依赖隐马尔可夫模型、最大熵模型和条件随机场等统计方法，它们共同推动了词性标注、命名实体识别等任务的发展。`
- 如果确实需要缩写：`条件随机场（CRF）常用于序列标注。后文直接使用 CRF。`

## 降低机器感

正文应避免：

- 过密的“第一层、第二层、第三层”机械罗列。
- 每段都用相同句式开头。
- 大量“本项目通过……实现……”重复句。
- 用附录、括号、术语堆叠凑字数。
- 将技术名词翻译、英文、缩写全部塞进同一句。

正文应优先：

- 用课程知识解释项目决策。
- 用真实实验数据、图表和调试过程支撑观点。
- 用自然段承接上下文。
- 将术语解释放在首次出现处，后续保持简洁。

## 交付前自检

生成报告后必须检查：

- 是否存在提示词式括号。
- 英文术语括号数量是否过多。
- 是否存在密集双语术语堆砌，导致正文像 AI 生成。
- 是否存在“待补充、此处、说明、可替换”等草稿词。
- 是否有过度模板化段落。
- 是否有重复长段或相似段落堆叠。
