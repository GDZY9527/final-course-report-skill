# DOCX 生成质量门槛

[English](docx_generation_quality.md) | [中文](docx_generation_quality.zh-CN.md)

## V3.0 回归问题

V3.0 测试暴露的问题：报告正文虽然改了字体字号，但目录仍沿用模板旧目录；章、节、条标题看起来像标题，实际 Word 样式仍是“正文/Normal”；标题段前段后过紧；旧模板结构没有被真正重建。

## 必须使用真实标题样式

生成 DOCX 时，所有章、节、条标题必须使用 Word 段落样式，而不是只修改普通正文段落的字体字号。

- 第一级（章）：必须使用模板中的“一级标题”或等价 Heading 1 样式。
- 第二级（节）：必须使用“二级标题”或等价 Heading 2 样式。
- 第三级（条）：必须使用“三/三级标题”或等价 Heading 3 样式。
- 第四级（项）：必须使用“四级标题”或等价项目标题样式；如模板无四级标题，按写作规范设置段落样式并记录说明。

禁止做法：

- `doc.add_paragraph()` 后只设置字体、字号、加粗或对齐，却不设置 `paragraph.style`。
- 用 `Normal` / “正文” 段落伪装标题。
- 手写标题后让目录无法识别。

## 目录必须重建

目录不得沿用模板旧目录。生成报告时必须：

1. 删除模板旧目录条目。
2. 根据新报告真实标题层级生成目录。
3. 如果能使用 Word 域，插入可刷新 TOC 域。
4. 如果不能刷新 Word 域，生成静态目录，但必须与新标题完全一致，并在最终说明中提醒用户在 Word 中刷新目录。

阻塞项：

- 目录中出现旧课程名、旧项目名、BERT、旧章节标题或模板残留。
- 导航窗格中标题不显示，或标题全部显示为正文样式。
- 实际标题与目录不一致。

## 段前段后

标题段前段后不得过紧。按 `course_report_writing_standard.zh-CN.md`：

- 章、节、条标题：段前 1 行，段后 1 行，单倍行距。
- 不要用 1pt、0pt 等视觉上紧贴正文的间距冒充“1 行”。
- 用 `python-docx` 时建议使用 `Pt(12)` 或与模板“一级标题/二级标题/三级标题”一致的段前段后值；不要使用 `space_before=1` 这种点值误写。

## 自动化写 DOCX 的推荐实现

使用 `python-docx` 时：

```python
p = doc.add_paragraph("第1章 XXX")
p.style = find_style(doc, ["一级标题", "Heading 1", "标题 1"])
apply_heading_font(p, level=1)
```

不要只写：

```python
p = doc.add_paragraph("第1章 XXX")
run = p.add_run(...)
run.font.size = Pt(18)
```

## 交付前检查

交付前必须检查：

- Word 导航窗格能识别一级、二级、三级标题。
- 目录内容来自新标题，不是模板旧目录。
- 标题段前段后符合规范。
- 正文字体字号、行距和缩进符合规范。
- 附录默认只有标题，正文留空。
