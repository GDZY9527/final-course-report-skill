# DOCX 生成质量门禁

[English](docx_generation_quality.md) | [中文](docx_generation_quality.zh-CN.md)

## V3/V4 回归问题

测试中反复出现的问题是：可见字体字号被修改了，但目录仍来自模板；章节标题看起来像标题，却仍是 `正文/Normal` 段落；标题上下间距过紧；最终目录仍保留旧项目名称。

## 必须使用真实标题样式

所有 DOCX 标题必须使用 Word 真实段落样式，而不是正文段落手工改字体。

- 章标题：使用模板中的 `一级标题` 或等价 Heading 1。
- 节标题：使用 `二级标题` 或等价 Heading 2。
- 小节标题：使用 `三级标题` 或等价 Heading 3。
- 条目标题：使用 `四级标题` 或等价条目标题样式；如果模板没有对应样式，按写作规范设置段落格式并记录限制。

禁止：

- 只调用 `doc.add_paragraph()` 后修改字体、字号、加粗或居中。
- 用 `正文/Normal` 样式伪装标题。
- 写出 Word 导航窗格和目录无法识别的标题。

## 必须重建目录

目录不得继承模板内容。智能体必须：

1. 记录模板中的目录段落。
2. 删除或替换模板目录段落。
3. 写完正文真实标题后，根据新标题层级生成目录。
4. 尽可能插入可刷新 Word 目录域。
5. 如果无法自动刷新目录域，生成与新标题完全一致的静态目录，并提醒用户在 Word 中刷新。

阻塞项：

- 目录中残留旧课程名、旧项目名、BERT、旧章节或模板痕迹。
- Word 导航窗格无法识别标题。
- 目录与正文实际标题不一致。
- 生成 DOCX 的目录文本与模板 DOCX 的目录文本相同。
- 最终 DOCX 保留旧目录域或旧目录可见文字。
- 交付根目录中同时留下多个报告 DOCX，且没有唯一最终版本。

不要把“用户打开 Word 后刷新目录”作为唯一解决方案。交付给用户时，文档可见目录不能仍是旧模板目录。

## 标题间距

标题不能视觉上过于紧凑。遵守 `course_report_writing_standard.zh-CN.md`：

- 章/节/小节标题：段前 1 行、段后 1 行、单倍行距。
- 不要把 1pt 或 0pt 间距伪装成“1 行”。
- 使用 `python-docx` 时优先使用 `Pt(12)` 或模板真实标题间距；不要把 `space_before=1` 误认为 1 行。

## 推荐 `python-docx` 模式

```python
p = doc.add_paragraph("第 1 章 XXX")
p.style = find_style(doc, ["一级标题", "Heading 1", "标题 1"])
apply_heading_font(p, level=1)
```

不要只写：

```python
p = doc.add_paragraph("第 1 章 XXX")
run = p.add_run(...)
run.font.size = Pt(18)
```

## 交付前检查

交付前必须验证：

- Word 导航窗格能识别一级、二级、三级标题。
- 目录来自新标题，而不是模板。
- 标题间距符合写作规范。
- 正文字体、字号、行距和缩进符合写作规范。
- 附录默认只有标题，没有正文填充。
- 交付根目录只保留一个最终报告 DOCX；旧草稿 DOCX 应移动到草稿目录并从最终打包中排除。
