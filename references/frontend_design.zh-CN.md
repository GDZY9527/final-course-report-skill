# 前端设计差异化

[English](frontend_design.md) | [中文](frontend_design.zh-CN.md)

## 先写设计简报

写前端代码前创建 `results/frontend_design_brief.md`，包含：

- 目标用户和主工作流
- 与项目绑定的信息架构
- 视觉风格和布局理由
- 与真实项目输出绑定的数据面板和交互
- 桌面端和移动端布局说明
- 加载、空状态、成功和错误状态

## 禁止统一仪表盘规则

skill 可以给出完整前端流程，但最终 UI 不能像复用模板。不得所有课程都交付同一种卡片网格/仪表盘结构。

每个前端必须有项目专属交互模型：

- C++ 系统：操作控制台、对象关系浏览器、状态迁移面板、编译/运行证据。
- NLP：样本分析工作台、词项/特征解释、预测对比、错误分析。
- 大模型应用：提示词/证据工作区、检索轨迹、回答质量评审、安全/溯源面板。
- 数据库：查询构造器、模式浏览器、事务/日志视图、数据质量面板。
- 物联网/软硬结合：设备/传感器监控、控制面板、告警时间线。

## 现代质量底线

前端应像经过打磨的课程项目，而不是赶工 demo：

- 领域专属导航和区块命名
- 有层级的排版、间距和字体
- 真实数据卡片、图表、表格、日志和详情面板
- 至少一个移动端断点
- 交互元素有 hover/focus/active 状态
- 明确加载、空状态、成功和错误状态
- 不做营销式大首屏；第一屏就是可用系统

配色应克制但有项目差异。避免一套蓝色仪表盘套所有项目。

## 框架门槛

Python Web 项目还必须读取 `frontend_framework.zh-CN.md`。默认使用 Django，除非用户明确指定其他技术栈。单文件 Flask/Jinja 页面不足以作为完整交付。

## 验证

在 `results/report_notes.md` 中记录前端证据：

- 打开的路由或文件
- 页面使用的数据源
- 截图等级：Level 1 真实浏览器截图或 Level 2 确定性预览
- 浏览器控制台是否有错误
- 移动端检查结果
- 已知限制
