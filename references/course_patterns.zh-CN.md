# 课程模式

[English](course_patterns.md) | [中文](course_patterns.zh-CN.md)

## C++ 面向对象程序设计

作为后续优化样例参考：

- C++17 核心项目
- 基于类的设计
- 文件持久化
- 测试与运行日志
- 类图和系统流程图
- 完整前端管理页面

模式：课程理论 + 面向对象系统 + 真实测试结果 + 前端可视化。

## 大模型应用开发

作为后续优化样例参考：

- RAG 或知识库问答项目
- API 路线和离线兜底
- 检索指标
- 前端问答页面
- 源码片段和报告备注

模式：保留模型/API 原计划，同时在环境受限时提供可复现本地兜底。

## 自然语言处理

仅作为早期历史样例参考：

- 理论章节结构
- 模型评价指标
- 降级说明
- 混淆矩阵和预测样例

不要复制它缺少前端的项目形态。新的 NLP 项目必须包含完整前后端，例如模型服务 API 和情感分析看板。

## 通用迁移方法

面对新课程：

1. 识别课程核心概念。
2. 选择一个小而完整、能体现课程概念的项目。
3. 除纯硬件课程外，补齐前后端。
4. 生成真实指标和截图。
5. 基于项目证据撰写第三章。
