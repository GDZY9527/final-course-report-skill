# 智能默认值

[English](smart_defaults.md) | [中文](smart_defaults.zh-CN.md)

## 目标

让 skill 能从很短的提示词开始工作。能安全推断的地方优先推断，只有缺失信息会阻塞任务或显著改变结果时才询问用户。

## 最小可用提示词

```text
使用这个 skill 为《{course_name}》生成期末结课报告。
模板：{template_docx}
Conda Python：{conda_python}
学生字段：{student_name}、{student_id}、{class}、{teacher}
```

可选：

```text
项目方向：……
目标课程目录：……
```

## 推断策略

先推断，再询问：

- 如果没有项目方向，选择一个紧凑、可交付、能体现课程核心概念且能产出真实结果的项目。
- 如果没有目标课程目录，在确认基础目录可写或经用户批准后，提出或创建 `{workspace}/{course_name}`。
- 如果缺少输出文件名，且学生字段已知，使用 `{course_name}-{student_id}-{student_name}.docx`。
- 如果课程不是明显纯硬件课程，必须包含完整前端。
- 如果高级技术栈不可用，选择本地基线，并把原计划路线写入后续优化。

仅在以下情况询问：

- 模板路径缺失且无法发现
- conda Python 路径缺失且需要真实运行
- 最终报告生成前缺少学生/封面字段
- 存在多个同样合理的课程目录或模板
- 课程可能是纯硬件，是否需要前端不明确

## 课程类型项目启发

- 程序设计/OOP：管理系统、仿真平台、迷你编译/解释工具、数据结构可视化、算法演示和前端看板。
- 数据库：CRUD 管理系统、分析看板、数据导入导出、查询可视化，数据库服务不可用时用 SQLite 兜底。
- AI/机器学习/NLP/CV：模型服务或基线流水线 + Web 演示、指标看板、预测样例、混淆矩阵或评价图。
- 大模型/智能体：RAG、工具调用智能体、Prompt 评价系统、知识库问答、离线兜底和前端页面。
- 软件工程：需求/任务/缺陷/项目管理系统，包含流程状态、图表和测试。
- 网络/安全：监控看板、数据包/日志分析器、规则模拟器、本地数据集兜底。
- 物联网/软硬结合：传感器数据模拟或硬件输入读取 + 监控/配置前端。
- 纯硬件：硬件设计、测试方案、测量结果、图纸和报告素材；除非存在软件监控/控制，否则前端可选。

## 命名默认值

需要迁移性时，项目文件夹使用易读 ASCII 名称：

- `{course_slug}_project`
- `{course_slug}_report_assets`
- `{course_slug}_delivery`

面向人的报告标题仍可使用中文课程名。

## 首次运行输出

使用推断默认值时，应清楚说明：

- 选择的项目方向
- 目标目录
- 前后端技术栈
- 降级方案
- 预计结果文件
