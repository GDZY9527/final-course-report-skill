# Originality And Sources

## Avoid Generic AI Prose

The report must sound like it comes from the real course project process.

Use:

- actual filenames
- actual module names
- real metrics and test counts
- real frontend behavior
- actual limitations encountered
- specific debugging or implementation observations
- personal reflection tied to course concepts

Avoid:

- empty slogans
- vague "I learned a lot" paragraphs
- repeated templates across courses
- invented problems or results
- exaggerated claims

## References

Include a references or sources section by default unless the template explicitly forbids it.

Use 5 to 8 stable sources when possible:

- course教材 or lecture materials if provided
- official documentation
- classic papers or technical specifications
- framework/library docs used by the project

Do not invent bibliographic details. If exact publication data is not known, use a simple source list with accurate titles and URLs or official names.
