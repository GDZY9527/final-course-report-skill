# Execution Log Template

[English](execution_log.md) | [中文](execution_log.zh-CN.md)

Maintain `results/execution_log.md` during each report run. This prevents the final retrospective from relying only on memory.

Required sections:

1. Agent name, model name if visible, platform, skill path, and date.
2. User prompts in chronological order.
3. Inputs collected or inferred.
4. Files read, created, modified, and intentionally not changed.
5. Commands run with working directory, important output, exit state, and generated files.
6. Failures, root causes, fixes, and whether each issue was fully resolved.
7. Verification results for project, frontend, screenshots, figures, DOCX, PDF, references, and packaging.
8. Limitations and truthful fallback notes.
9. Final deliverables.
10. Suggested improvements to the skill.

Do not include secrets, API keys, private tokens, or unnecessary personal data.
