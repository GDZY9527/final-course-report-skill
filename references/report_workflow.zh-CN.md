# 报告工作流

[English](report_workflow.md) | [中文](report_workflow.zh-CN.md)

## 目的

本流程用于根据课程名称、项目方向、Word 模板和目标运行环境，生成完整期末结课报告及其配套实践项目。项目方向是最小必需输入：可以是用户明确指定的项目，也可以明确写“暂无明确项目目标，请智能体自行设计唯一性项目”。目标目录是可选覆盖项。

## 预检查

收集以下信息：

- 课程名称
- 项目方向，或明确授权智能体自行设计唯一性项目
- 可选目标课程目录
- 模板 `.docx`
- conda Python 路径
- 姓名、学号、班级、教师和要求的输出文件名
- 是否为纯硬件课程

能从本地环境发现的信息先自行检查，不要过早询问用户。

## 两阶段生成

1. **项目阶段**
   - 创建真实可运行项目。
   - 生成 README、requirements、运行脚本、源码、前端、后端/核心逻辑、数据、测试、结果文件、截图和报告备注。
   - 使用 `{conda_python}` 运行或记录验证命令。

2. **报告阶段**
   - 写作前先分析 Word 模板。
   - 只保留模板中有价值的结构，例如封面字段、目录区域、页眉页脚位置、附录标题和评阅表。
   - 字体、字号、标题、页边距、页眉页脚、图题表题、公式和代码格式必须以 `course_report_writing_standard.zh-CN.md` 为准。
   - 基于真实项目输出写作。
   - 有文档工具时进行渲染和视觉检查。

## 默认报告结构

- 封面
- 目录
- 第一章：课程理论基础
- 第二章：核心技术与方法
- 第三章：项目实践
- 第四章：心得收获与后续改进
- 参考文献或资料来源
- 附录

## 写作要求

- 使用正式但自然的中文学术表达。
- 关键理论小节应连接概念、原理、工程意义和项目联系。
- 每张图和每个表格都要有正文解读。
- 第三章必须说明需求背景、架构、数据流、前端交互、后端/核心逻辑、测试、结果、限制和优化方向。
- 不复制其他课程报告正文，只复用结构和质量标准。

## 最终答复清单

最终说明中列出：

- Word 和 PDF 路径
- 项目路径
- 使用的 conda 环境路径
- 图表清单
- 验证摘要
- 已知限制
- 待确认的打包清单或最终交付包路径
